# Túneis

Um túnel é um link público para o seu CodeCaptain, para que você possa acessá-lo pelo celular ou de outra rede. Use `codecaptain tunnel` para criar um para uma instância em execução.

## Pré-requisitos

O CodeCaptain inicia o CLI do provedor de túnel na sua máquina. Instale primeiro o provedor que você quer usar:

```bash
brew install cloudflared
brew install ngrok
```

Os túneis rápidos da Cloudflare funcionam com `cloudflared`. O Ngrok exige uma conta ngrok e um authtoken do painel do ngrok:

```bash
ngrok config add-authtoken <your-ngrok-token>
```

## Início rápido

1. Inicie o CodeCaptain:

```bash
codecaptain
```

Se você pular esta etapa, `codecaptain tunnel start` pode iniciar automaticamente um servidor CLI. Ao iniciar automaticamente, você pode passar opções do servidor como `--port`, `--host`, `--lan`, `--ui-password` e `--api-only`.

2. Inicie um túnel da Cloudflare:

```bash
codecaptain tunnel start --provider cloudflare --mode quick
```

Ou inicie um túnel do Ngrok:

```bash
codecaptain tunnel start --provider ngrok --mode quick
```

3. Verifique o status:

```bash
codecaptain tunnel status
```

Quando o túnel estiver no ar, o `status` mostra uma URL pública. Abra-a, ou escaneie o QR code, para acessar o CodeCaptain de qualquer lugar.

Por padrão, o CodeCaptain imprime um QR code em sessões TTY interativas. Use `--qr` para forçar a saída do QR, ou `--no-qr` para desativá-la.

## Provedores

- `cloudflare`: modos quick, managed remote e managed local
- `ngrok`: modo quick

## Modos gerenciados

### Gerenciado remoto

Use um token + nome de host gerenciados pela Cloudflare:

```bash
codecaptain tunnel start --provider cloudflare --mode managed-remote --token-file ~/.secrets/cf-token --hostname app.example.com
```

### Gerenciado local

Use uma configuração local do `cloudflared`:

```bash
codecaptain tunnel start --provider cloudflare --mode managed-local --config ~/.cloudflared/config.yml
```

## Perfis (managed-remote)

Salve um perfil reutilizável:

```bash
codecaptain tunnel profile add --provider cloudflare --mode managed-remote --name prod-main --hostname app.example.com --token-file ~/.secrets/cf-token
```

Inicie usando o perfil salvo:

```bash
codecaptain tunnel start --profile prod-main
```

## Comandos úteis

```bash
codecaptain tunnel providers
codecaptain tunnel ready --provider cloudflare
codecaptain tunnel ready --provider ngrok
codecaptain tunnel doctor --provider cloudflare
codecaptain tunnel doctor --provider ngrok
codecaptain tunnel stop --port 3000
```

## Notas de comportamento

- um único túnel ativo por instância do CodeCaptain (porta)
- iniciar um novo modo/provedor na mesma instância substitui o túnel anterior
- gerar um novo link de conexão revoga o anterior não utilizado
- o auto-start do túnel preserva flags do servidor como `--ui-password` e `--api-only` nas configurações da instância usadas por reinícios e atualizações
