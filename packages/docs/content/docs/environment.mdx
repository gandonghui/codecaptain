# Environment Variables

CodeCaptain reads these environment variables at startup. For startup services, `codecaptain startup enable` snapshots the current environment by default, so rerun it after changing variables you want the service to use.

## CodeCaptain server

### `CODECAPTAIN_HOST`

Bind address for the CodeCaptain web server. Use `0.0.0.0` to allow access from other machines.

### `CODECAPTAIN_UI_PASSWORD`

Password for the browser UI. Use this when binding outside localhost, using tunnels, or running behind a reverse proxy.

### `CODECAPTAIN_API_ONLY`

Starts CodeCaptain in headless mode when set to `true` or `1`. API routes stay available for desktop and mobile clients, but the browser UI is not served.

### `CODECAPTAIN_DATA_DIR`

Overrides the CodeCaptain data directory. The default is `~/.config/codecaptain`.

### `CODECAPTAIN_COMPRESS_API`

Controls API response compression. Use `true` or `1` to force it on, `false` or `0` to force it off.

### `CODECAPTAIN_SKIP_API_COMPRESSION`

Disables API response compression when set to `true` or `1`. This takes precedence over `CODECAPTAIN_COMPRESS_API`.

### `CODECAPTAIN_VERBOSE_REQUEST_LOGS`

Enables verbose HTTP request logs when set to `true` or `1`.

### `CODECAPTAIN_UPDATE_API_URL`

Overrides the update-check API endpoint. Most users should leave this unset.

### `CODECAPTAIN_PACKAGE_MANAGER`

Forces the package manager used by update operations when auto-detection is wrong.

## CodeCaptain-core server

### `OPENCODE_HOST`

Connects CodeCaptain to an existing CodeCaptain-core server. The value must be an `http` or `https` origin with an explicit port and no path, query, or hash. `OPENCODE_HOST` takes precedence over `OPENCODE_PORT`.

### `OPENCODE_PORT`

Sets the CodeCaptain-core server port. With managed CodeCaptain-core this requests the managed port; with `OPENCODE_SKIP_START=true` it connects to an external server on that port.

### `OPENCODE_SKIP_START`

Prevents CodeCaptain from starting its own CodeCaptain-core server when set to `true`.

### `CODECAPTAIN_OPENCODE_HOSTNAME`

Bind hostname for the CodeCaptain-core server managed by CodeCaptain. The default is `127.0.0.1`.

### `OPENCODE_BINARY`

Path to the `opencode` executable CodeCaptain should run.

### `OPENCODE_CONFIG`

Path to a specific CodeCaptain-core config file.

### `OPENCODE_CONFIG_DIR`

Path to a specific CodeCaptain-core config directory for agents, skills, snippets, and config discovery.

### `OPENCODE_DATA_DIR`

Custom data directory for the managed CodeCaptain-core server.

### `OPENCODE_WSL_DISTRO`

Selects the WSL distro used for CodeCaptain-core integration on Windows.

### `CODECAPTAIN_OPENCODE_WSL_DISTRO`

CodeCaptain-specific alias for selecting the WSL distro. `OPENCODE_WSL_DISTRO` takes precedence when both are set.

### `OPENCODE_JWT_SECRET`

Secret used to sign UI authentication tokens. Use a long random value for persistent service deployments.

## Terminal and Git

### `CODECAPTAIN_TERMINAL_SHELL`

Shell executable used for CodeCaptain terminal sessions.

### `CODECAPTAIN_GIT_BINARY`

Git executable used by CodeCaptain Git features.

### `GIT_BINARY`

Alternative Git executable override. Prefer `CODECAPTAIN_GIT_BINARY` for CodeCaptain-specific configuration.

### `CODECAPTAIN_GIT_READ_CACHE_TTL_MS`

Time-to-live, in milliseconds, for cached Git-backed file reads. Set `0` to disable this cache while debugging.

## Voice and tunnels

### `OPENAI_API_KEY`

API key used by CodeCaptain voice features that call OpenAI-compatible services.

### `CODECAPTAIN_ALLOW_REMOTE_OPENAI_COMPAT_URLS`

Allows remote OpenAI-compatible base URLs for voice features when set to `true` or `1`.

### `NGROK_AUTHTOKEN`

ngrok auth token used by CodeCaptain tunnel commands. You can also configure ngrok with `ngrok config add-authtoken <token>`.

## Runtime helpers

### `BUN_BINARY`

Bun executable CodeCaptain should use when spawning daemon processes.

### `BUN_INSTALL`

Bun installation root. CodeCaptain uses it to find `bin/bun` for daemon startup and updates.

### `VITE_OPENCODE_URL`

Build-time API base URL for the Vite-built web app. Most users should not set it for normal CLI or desktop usage.
