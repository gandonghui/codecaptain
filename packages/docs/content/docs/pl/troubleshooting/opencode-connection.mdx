# Połączenie z CodeCaptain-core

Jeśli CodeCaptain się ładuje, ale nigdy nie przechodzi poza „CodeCaptain-core uruchamia się ponownie”, albo czat nie odpowiada, serwer, z którym się komunikuje, jest nieosiągalny. Przejdź przez te kroki.

## Utknął na „CodeCaptain-core uruchamia się ponownie”

- daj mu chwilę tuż po uruchomieniu — ten stan jest normalny, gdy serwer startuje
- sprawdź, czy serwer żyje, za pomocą `codecaptain status`
- zrestartuj go za pomocą `codecaptain restart`
- zobacz szczegóły startu za pomocą `codecaptain logs`

## Łączysz się z własnym serwerem

Jeśli ustawiłeś CodeCaptain tak, aby używał istniejącego serwera, sprawdź ponownie konfigurację z [CodeCaptain-core Server](/pl/opencode-server/):

- `OPENCODE_HOST` musi zawierać port, bez ścieżki — na przykład `http://localhost:4096`
- ustaw `OPENCODE_SKIP_START=true`, aby CodeCaptain nie uruchamiał też własnego
- jeśli adres jest nieprawidłowy, CodeCaptain go ignoruje i uruchamia własny serwer — poszukaj ostrzeżenia `[config]` w logach

## Nadal nie działa

- potwierdź, że Node.js jest w wersji `20` lub nowszej
- przeinstaluj najnowsze CLI
- otwórz `http://localhost:3000` bezpośrednio, zanim przetestujesz jakikolwiek tunel lub zdalny link
