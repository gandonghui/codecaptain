# Zmienne środowiskowe

CodeCaptain odczytuje te zmienne przy starcie. Dla usług startowych `codecaptain startup enable` domyślnie zapisuje snapshot bieżącego środowiska, więc uruchom tę komendę ponownie po zmianie zmiennych, których ma używać usługa.

## Serwer CodeCaptain

### `CODECAPTAIN_HOST`

Adres, na którym nasłuchuje serwer web CodeCaptain. Użyj `0.0.0.0`, aby pozwolić na dostęp z innych maszyn.

### `CODECAPTAIN_UI_PASSWORD`

Hasło do interfejsu w przeglądarce. Ustaw je przy dostępie spoza localhost, tunelach albo reverse proxy.

### `CODECAPTAIN_API_ONLY`

Uruchamia CodeCaptain w trybie headless, gdy ustawione na `true` lub `1`. Trasy API pozostają dostępne dla klientów desktopowych i mobilnych, ale UI przeglądarki nie jest serwowane.

### `CODECAPTAIN_DATA_DIR`

Nadpisuje katalog danych CodeCaptain. Domyślnie jest to `~/.config/codecaptain`.

### `CODECAPTAIN_COMPRESS_API`

Steruje kompresją odpowiedzi API. `true` albo `1` ją włącza, `false` albo `0` wyłącza.

### `CODECAPTAIN_SKIP_API_COMPRESSION`

Wyłącza kompresję odpowiedzi API, gdy ma wartość `true` albo `1`. Ma pierwszeństwo przed `CODECAPTAIN_COMPRESS_API`.

### `CODECAPTAIN_VERBOSE_REQUEST_LOGS`

Włącza szczegółowe HTTP request logs, gdy ma wartość `true` albo `1`.

### `CODECAPTAIN_UPDATE_API_URL`

Nadpisuje endpoint sprawdzania aktualizacji. Większość użytkowników nie powinna tego ustawiać.

### `CODECAPTAIN_PACKAGE_MANAGER`

Wymusza package manager używany do aktualizacji, gdy automatyczne wykrywanie się pomyli.

## Serwer CodeCaptain-core

### `OPENCODE_HOST`

Łączy CodeCaptain z istniejącym CodeCaptain-core server. Wartość musi być origin `http` albo `https` z jawnym portem i bez path, query ani hash. `OPENCODE_HOST` ma pierwszeństwo przed `OPENCODE_PORT`.

### `OPENCODE_PORT`

Ustawia port CodeCaptain-core server. Dla managed CodeCaptain-core prosi o ten port; z `OPENCODE_SKIP_START=true` łączy się z zewnętrznym serwerem na tym porcie.

### `OPENCODE_SKIP_START`

Gdy ma wartość `true`, blokuje uruchamianie własnego CodeCaptain-core server przez CodeCaptain.

### `CODECAPTAIN_OPENCODE_HOSTNAME`

Hostname dla CodeCaptain-core server zarządzanego przez CodeCaptain. Domyślnie `127.0.0.1`.

### `OPENCODE_BINARY`

Ścieżka do executable `opencode`, którego ma używać CodeCaptain.

### `OPENCODE_CONFIG`

Ścieżka do konkretnego pliku config CodeCaptain-core.

### `OPENCODE_CONFIG_DIR`

Ścieżka do katalogu config CodeCaptain-core dla agents, skills, snippets i config discovery.

### `OPENCODE_DATA_DIR`

Własny katalog danych dla managed CodeCaptain-core server.

### `OPENCODE_WSL_DISTRO`

Wybiera WSL distro używane do integracji CodeCaptain-core w Windows.

### `CODECAPTAIN_OPENCODE_WSL_DISTRO`

Alias specyficzny dla CodeCaptain do wyboru WSL distro. `OPENCODE_WSL_DISTRO` ma pierwszeństwo, gdy ustawione są obie zmienne.

### `OPENCODE_JWT_SECRET`

Secret używany do podpisywania UI auth tokens. Dla trwałych usług użyj długiej losowej wartości.

## Terminal i Git

### `CODECAPTAIN_TERMINAL_SHELL`

Shell executable używany przez terminal sessions w CodeCaptain.

### `CODECAPTAIN_GIT_BINARY`

Git executable używany przez funkcje Git w CodeCaptain.

### `GIT_BINARY`

Alternatywny override dla Git executable. Dla konfiguracji specyficznej dla CodeCaptain preferuj `CODECAPTAIN_GIT_BINARY`.

### `CODECAPTAIN_GIT_READ_CACHE_TTL_MS`

TTL w milisekundach dla cached Git-backed file reads. Ustaw `0`, aby wyłączyć cache podczas debugowania.

## Głos i tunele

### `OPENAI_API_KEY`

API key używany przez funkcje głosowe CodeCaptain, które wywołują OpenAI-compatible services.

### `CODECAPTAIN_ALLOW_REMOTE_OPENAI_COMPAT_URLS`

Pozwala na zdalne OpenAI-compatible base URLs dla funkcji głosowych, gdy ma wartość `true` albo `1`.

### `NGROK_AUTHTOKEN`

ngrok auth token używany przez tunnel commands CodeCaptain. Możesz też skonfigurować go przez `ngrok config add-authtoken <token>`.

## Runtime helpers

### `BUN_BINARY`

Bun executable, którego CodeCaptain ma używać do uruchamiania daemon processes.

### `BUN_INSTALL`

Katalog główny instalacji Bun. CodeCaptain używa go, aby znaleźć `bin/bun` dla daemon startup i aktualizacji.

### `VITE_OPENCODE_URL`

Build-time API base URL dla Vite-built web app. Przy normalnym użyciu CLI lub desktop większość użytkowników nie powinna tego ustawiać.
