# Updates

How you update CodeCaptain depends on how you installed it. In every case you can check your current version at **Settings → CodeCaptain → About**.

## Desktop app

The desktop app checks for updates against GitHub releases. When one is available, CodeCaptain tells you, downloads it when you choose, and installs it on the next restart. You're always in control — nothing installs without your say-so.

## Web / CLI

If you installed the CLI, update it from **About** with the check and update buttons, or from the terminal:

```bash
codecaptain update
```

CodeCaptain detects how it was installed (npm, pnpm, yarn, or bun) and runs the right update for you.

## The CodeCaptain-core server

CodeCaptain and CodeCaptain-core update separately. When a new CodeCaptain-core version is available, CodeCaptain offers to update it and restarts the server afterward. See [CodeCaptain-core Server](/opencode-server/) for how that server is managed.
