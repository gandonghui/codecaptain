# CodeCaptain-core Connection

If CodeCaptain loads but never gets past "CodeCaptain-core is restarting," or chat won't respond, the server it talks to isn't reachable. Work through these.

## It's stuck on "CodeCaptain-core is restarting"

- give it a moment right after launch — this state is normal while the server starts
- check whether the server is alive with `codecaptain status`
- restart it with `codecaptain restart`
- view startup details with `codecaptain logs`

## You're connecting to your own server

If you set CodeCaptain to use an existing server, double-check the setup from [CodeCaptain-core Server](/opencode-server/):

- `OPENCODE_HOST` must include the port, with no path — for example `http://localhost:4096`
- set `OPENCODE_SKIP_START=true` so CodeCaptain doesn't also start its own
- if the address is invalid, CodeCaptain ignores it and starts its own server instead — look for a `[config]` warning in the logs

## Still failing

- confirm Node.js is version `20` or newer
- reinstall the latest CLI
- open `http://localhost:3000` directly before testing any tunnel or remote link
