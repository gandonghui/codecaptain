# Змінні середовища

CodeCaptain читає ці змінні під час запуску. Для startup-сервісів `codecaptain startup enable` типово зберігає snapshot поточного середовища, тому після змін перезапустіть цю команду.

## Сервер CodeCaptain

### `CODECAPTAIN_HOST`

Адреса, на якій слухає web-сервер CodeCaptain. Використовуйте `0.0.0.0`, щоб відкрити доступ з інших машин.

### `CODECAPTAIN_UI_PASSWORD`

Пароль для browser UI. Використовуйте його для доступу не лише з localhost, тунелів або reverse proxy.

### `CODECAPTAIN_API_ONLY`

Запускає CodeCaptain у headless mode, якщо встановлено `true` або `1`. API routes залишаються доступними для desktop і mobile clients, але browser UI не віддається.

### `CODECAPTAIN_DATA_DIR`

Перевизначає директорію даних CodeCaptain. Типово це `~/.config/codecaptain`.

### `CODECAPTAIN_COMPRESS_API`

Керує стисненням API-відповідей. `true` або `1` вмикає, `false` або `0` вимикає.

### `CODECAPTAIN_SKIP_API_COMPRESSION`

Вимикає стиснення API-відповідей, якщо встановлено `true` або `1`. Має пріоритет над `CODECAPTAIN_COMPRESS_API`.

### `CODECAPTAIN_VERBOSE_REQUEST_LOGS`

Вмикає детальні HTTP request logs, якщо встановлено `true` або `1`.

### `CODECAPTAIN_UPDATE_API_URL`

Перевизначає endpoint для перевірки оновлень. Більшості користувачів це не потрібно.

### `CODECAPTAIN_PACKAGE_MANAGER`

Примусово задає package manager для оновлень, якщо auto-detection помилився.

## Сервер CodeCaptain-core

### `OPENCODE_HOST`

Підключає CodeCaptain до вже запущеного CodeCaptain-core server. Значення має бути `http` або `https` origin з явним портом і без path, query або hash. `OPENCODE_HOST` має пріоритет над `OPENCODE_PORT`.

### `OPENCODE_PORT`

Задає порт CodeCaptain-core server. Для managed CodeCaptain-core це бажаний порт; з `OPENCODE_SKIP_START=true` CodeCaptain підключається до зовнішнього сервера на цьому порту.

### `OPENCODE_SKIP_START`

Не дає CodeCaptain запускати власний CodeCaptain-core server, якщо встановлено `true`.

### `CODECAPTAIN_OPENCODE_HOSTNAME`

Hostname для CodeCaptain-core server, яким керує CodeCaptain. Типово `127.0.0.1`.

### `OPENCODE_BINARY`

Шлях до executable `opencode`, який має запускати CodeCaptain.

### `OPENCODE_CONFIG`

Шлях до конкретного config-файлу CodeCaptain-core.

### `OPENCODE_CONFIG_DIR`

Шлях до директорії config CodeCaptain-core для agents, skills, snippets і config discovery.

### `OPENCODE_DATA_DIR`

Власна директорія даних для managed CodeCaptain-core server.

### `OPENCODE_WSL_DISTRO`

Вибирає WSL distro для інтеграції CodeCaptain-core на Windows.

### `CODECAPTAIN_OPENCODE_WSL_DISTRO`

CodeCaptain-specific alias для вибору WSL distro. `OPENCODE_WSL_DISTRO` має пріоритет, якщо задані обидві змінні.

### `OPENCODE_JWT_SECRET`

Secret для підпису UI auth tokens. Для постійних service deployments використовуйте довге випадкове значення.

## Terminal і Git

### `CODECAPTAIN_TERMINAL_SHELL`

Shell executable для terminal sessions в CodeCaptain.

### `CODECAPTAIN_GIT_BINARY`

Git executable для Git-функцій CodeCaptain.

### `GIT_BINARY`

Альтернативний override для Git executable. Для CodeCaptain-specific налаштувань краще використовуйте `CODECAPTAIN_GIT_BINARY`.

### `CODECAPTAIN_GIT_READ_CACHE_TTL_MS`

TTL у мілісекундах для cached Git-backed file reads. Встановіть `0`, щоб вимкнути cache під час debugging.

## Voice і tunnels

### `OPENAI_API_KEY`

API key для voice-функцій CodeCaptain, які викликають OpenAI-compatible services.

### `CODECAPTAIN_ALLOW_REMOTE_OPENAI_COMPAT_URLS`

Дозволяє remote OpenAI-compatible base URLs для voice-функцій, якщо встановлено `true` або `1`.

### `NGROK_AUTHTOKEN`

ngrok auth token для tunnel-команд CodeCaptain. Також можна налаштувати ngrok через `ngrok config add-authtoken <token>`.

## Runtime helpers

### `BUN_BINARY`

Bun executable, який CodeCaptain має використовувати для запуску daemon processes.

### `BUN_INSTALL`

Корінь інсталяції Bun. CodeCaptain використовує його, щоб знайти `bin/bun` для daemon startup і updates.

### `VITE_OPENCODE_URL`

Build-time API base URL для Vite-built web app. Для звичайного CLI або desktop usage більшості користувачів це не потрібно.
