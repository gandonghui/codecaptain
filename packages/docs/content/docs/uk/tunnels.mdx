# Тунелі

Тунель — це публічне посилання на ваш CodeCaptain, щоб дістатися до нього з телефона чи з іншої мережі. Скористайтеся `codecaptain tunnel`, щоб створити його для запущеного інстансу.

## Передумови

CodeCaptain запускає CLI провайдера тунелю на вашій машині. Спочатку встановіть провайдера, який хочете використовувати:

```bash
brew install cloudflared
brew install ngrok
```

Швидкі тунелі Cloudflare працюють через `cloudflared`. Для Ngrok потрібен акаунт ngrok і authtoken з dashboard ngrok:

```bash
ngrok config add-authtoken <your-ngrok-token>
```

## Швидкий старт

1. Запустіть CodeCaptain:

```bash
codecaptain
```

Якщо пропустити цей крок, `codecaptain tunnel start` може автоматично запустити CLI server. Під час auto-start можна передати server options: `--port`, `--host`, `--lan`, `--ui-password` і `--api-only`.

2. Запустіть тунель Cloudflare:

```bash
codecaptain tunnel start --provider cloudflare --mode quick
```

Або запустіть тунель Ngrok:

```bash
codecaptain tunnel start --provider ngrok --mode quick
```

3. Перевірте статус:

```bash
codecaptain tunnel status
```

Коли тунель піднято, `status` показує публічний URL. Відкрийте його або відскануйте QR-код, щоб дістатися до CodeCaptain звідусіль.

Типово CodeCaptain виводить QR-код в інтерактивних TTY-сесіях. Використовуйте `--qr`, щоб примусово вивести QR, або `--no-qr`, щоб вимкнути його.

## Провайдери

- `cloudflare`: quick, managed remote і managed local режими
- `ngrok`: quick режим

## Керовані режими

### Кероване віддалене

Використовуйте токен + хостнейм, керовані Cloudflare:

```bash
codecaptain tunnel start --provider cloudflare --mode managed-remote --token-file ~/.secrets/cf-token --hostname app.example.com
```

### Кероване локальне

Використовуйте локальний конфіг `cloudflared`:

```bash
codecaptain tunnel start --provider cloudflare --mode managed-local --config ~/.cloudflared/config.yml
```

## Профілі (managed-remote)

Збережіть багаторазовий профіль:

```bash
codecaptain tunnel profile add --provider cloudflare --mode managed-remote --name prod-main --hostname app.example.com --token-file ~/.secrets/cf-token
```

Запуск зі збереженим профілем:

```bash
codecaptain tunnel start --profile prod-main
```

## Корисні команди

```bash
codecaptain tunnel providers
codecaptain tunnel ready --provider cloudflare
codecaptain tunnel ready --provider ngrok
codecaptain tunnel doctor --provider cloudflare
codecaptain tunnel doctor --provider ngrok
codecaptain tunnel stop --port 3000
```

## Нотатки щодо поведінки

- один активний тунель на інстанс CodeCaptain (порт)
- запуск нового режиму/провайдера на тому самому інстансі замінює попередній тунель
- генерація нового посилання для підключення відкликає попереднє невикористане
