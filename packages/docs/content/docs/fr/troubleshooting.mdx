# Dépannage

Vous êtes bloqué ? Trouvez le symptôme ci-dessous et essayez la correction.

## La commande CodeCaptain se ferme ou ne démarre pas

- confirmez que Node.js est en version `>=22`
- lancez `codecaptain --version`
- réinstallez la dernière CLI si nécessaire

## L’UI web n’est pas joignable

- consultez les logs serveur avec `codecaptain logs`
- vérifiez le port actif (par défaut `3000`)
- ouvrez d’abord `http://localhost:3000` directement avant de tester les liens de tunnel

## Le lien distant/tunnel ne fonctionne pas

- lancez `codecaptain tunnel status --all`
- redémarrez le tunnel depuis la même instance/le même port
- régénérez le lien de connexion si le token précédent a déjà été utilisé

Voir [Tunnels](/tunnels/) pour la configuration complète, et [Reverse proxy](/reverse-proxy/) si vous en utilisez un.

## L’extension VS Code ne se connecte pas

- confirmez que le serveur CodeCaptain tourne
- vérifiez que l’extension est à jour
- rechargez la fenêtre VS Code et réessayez la connexion
