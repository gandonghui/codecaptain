# Connexion à CodeCaptain-core

Si CodeCaptain se charge mais reste bloqué sur « CodeCaptain-core redémarre », ou si le chat ne répond pas, le serveur auquel il parle n’est pas joignable. Suivez ces vérifications.

## Il reste bloqué sur « CodeCaptain-core redémarre »

- laissez-lui un moment juste après le lancement — cet état est normal pendant le démarrage du serveur
- vérifiez si le serveur est vivant avec `codecaptain status`
- redémarrez-le avec `codecaptain restart`
- consultez les détails de démarrage avec `codecaptain logs`

## Vous vous connectez à votre propre serveur

Si vous avez configuré CodeCaptain pour utiliser un serveur existant, revérifiez la configuration depuis [Serveur CodeCaptain-core](/opencode-server/) :

- `OPENCODE_HOST` doit inclure le port, sans chemin — par exemple `http://localhost:4096`
- définissez `OPENCODE_SKIP_START=true` pour qu’CodeCaptain ne démarre pas aussi le sien
- si l’adresse est invalide, CodeCaptain l’ignore et démarre son propre serveur à la place — cherchez un avertissement `[config]` dans les logs

## Toujours en échec

- confirmez que Node.js est en version `20` ou plus récente
- réinstallez la dernière CLI
- ouvrez `http://localhost:3000` directement avant de tester un tunnel ou lien distant
