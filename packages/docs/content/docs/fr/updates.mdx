# Mises à jour

La façon de mettre CodeCaptain à jour dépend de votre méthode d’installation. Dans tous les cas, vous pouvez vérifier votre version actuelle dans **Paramètres → CodeCaptain → À propos**.

## Application desktop

L’application desktop vérifie les mises à jour via les releases GitHub. Quand une mise à jour est disponible, CodeCaptain vous le signale, la télécharge lorsque vous le choisissez et l’installe au prochain redémarrage. Vous gardez toujours le contrôle — rien ne s’installe sans votre accord.

## Web / CLI

Si vous avez installé la CLI, mettez-la à jour depuis **À propos** avec les boutons de vérification et de mise à jour, ou depuis le terminal :

```bash
codecaptain update
```

CodeCaptain détecte comment il a été installé (npm, pnpm, yarn ou bun) et lance la bonne mise à jour pour vous.

## Le serveur CodeCaptain-core

CodeCaptain et CodeCaptain-core se mettent à jour séparément. Quand une nouvelle version d’CodeCaptain-core est disponible, CodeCaptain propose de la mettre à jour puis redémarre le serveur. Voir [Serveur CodeCaptain-core](/opencode-server/) pour comprendre comment ce serveur est géré.
