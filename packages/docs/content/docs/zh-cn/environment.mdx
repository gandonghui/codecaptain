# 环境变量

CodeCaptain 会在启动时读取这些变量。对于开机启动服务，`codecaptain startup enable` 默认会保存当前环境的 snapshot，所以如果要让服务使用新的变量，请重新运行它。

## CodeCaptain 服务器

### `CODECAPTAIN_HOST`

CodeCaptain web 服务器监听的地址。使用 `0.0.0.0` 可允许其他机器访问。

### `CODECAPTAIN_UI_PASSWORD`

浏览器 UI 的密码。当你绑定到 localhost 之外、使用隧道或反向代理时，请设置它。

### `CODECAPTAIN_API_ONLY`

设置为 `true` 或 `1` 时，以 headless 模式启动 CodeCaptain。桌面和移动客户端仍可使用 API 路由，但不会提供浏览器 UI。

### `CODECAPTAIN_DATA_DIR`

覆盖 CodeCaptain 数据目录。默认是 `~/.config/codecaptain`。

### `CODECAPTAIN_COMPRESS_API`

控制 API 响应压缩。`true` 或 `1` 强制开启，`false` 或 `0` 强制关闭。

### `CODECAPTAIN_SKIP_API_COMPRESSION`

设置为 `true` 或 `1` 时关闭 API 响应压缩。它优先于 `CODECAPTAIN_COMPRESS_API`。

### `CODECAPTAIN_VERBOSE_REQUEST_LOGS`

设置为 `true` 或 `1` 时启用详细 HTTP request logs。

### `CODECAPTAIN_UPDATE_API_URL`

覆盖更新检查 API endpoint。大多数用户不需要设置它。

### `CODECAPTAIN_PACKAGE_MANAGER`

当自动检测错误时，强制指定更新操作使用的 package manager。

## CodeCaptain-core 服务器

### `OPENCODE_HOST`

将 CodeCaptain 连接到已有的 CodeCaptain-core server。值必须是带明确端口、没有 path、query 或 hash 的 `http` 或 `https` origin。`OPENCODE_HOST` 优先于 `OPENCODE_PORT`。

### `OPENCODE_PORT`

设置 CodeCaptain-core server 端口。对于 managed CodeCaptain-core，它请求使用该端口；配合 `OPENCODE_SKIP_START=true` 时，它连接到该端口上的外部服务器。

### `OPENCODE_SKIP_START`

设置为 `true` 时阻止 CodeCaptain 启动自己的 CodeCaptain-core server。

### `CODECAPTAIN_OPENCODE_HOSTNAME`

CodeCaptain 管理的 CodeCaptain-core server 绑定 hostname。默认是 `127.0.0.1`。

### `OPENCODE_BINARY`

CodeCaptain 应该运行的 `opencode` executable 路径。

### `OPENCODE_CONFIG`

指定 CodeCaptain-core config 文件路径。

### `OPENCODE_CONFIG_DIR`

指定 CodeCaptain-core config 目录，用于 agents、skills、snippets 和 config discovery。

### `OPENCODE_DATA_DIR`

managed CodeCaptain-core server 的自定义数据目录。

### `OPENCODE_WSL_DISTRO`

选择 Windows 上 CodeCaptain-core 集成使用的 WSL distro。

### `CODECAPTAIN_OPENCODE_WSL_DISTRO`

CodeCaptain 专用的 WSL distro alias。如果两个变量都设置，`OPENCODE_WSL_DISTRO` 优先。

### `OPENCODE_JWT_SECRET`

用于签名 UI auth tokens 的 secret。持久服务部署应使用长的随机值。

## Terminal 和 Git

### `CODECAPTAIN_TERMINAL_SHELL`

CodeCaptain terminal sessions 使用的 shell executable。

### `CODECAPTAIN_GIT_BINARY`

CodeCaptain Git 功能使用的 Git executable。

### `GIT_BINARY`

Git executable 的备用 override。CodeCaptain 专用配置建议使用 `CODECAPTAIN_GIT_BINARY`。

### `CODECAPTAIN_GIT_READ_CACHE_TTL_MS`

Git-backed file reads cache 的 TTL，单位为毫秒。调试时可设置为 `0` 禁用 cache。

## 语音和隧道

### `OPENAI_API_KEY`

CodeCaptain 语音功能调用 OpenAI-compatible services 时使用的 API key。

### `CODECAPTAIN_ALLOW_REMOTE_OPENAI_COMPAT_URLS`

设置为 `true` 或 `1` 时，允许语音功能使用远程 OpenAI-compatible base URLs。

### `NGROK_AUTHTOKEN`

CodeCaptain tunnel commands 使用的 ngrok auth token。也可以用 `ngrok config add-authtoken <token>` 配置。

## Runtime helpers

### `BUN_BINARY`

CodeCaptain 启动 daemon processes 时使用的 Bun executable。

### `BUN_INSTALL`

Bun 安装根目录。CodeCaptain 使用它查找 `bin/bun` 来启动 daemon 和执行更新。

### `VITE_OPENCODE_URL`

Vite-built web app 的 build-time API base URL。普通 CLI 或 desktop 使用通常不需要设置它。
