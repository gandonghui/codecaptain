# Tunnels

A tunnel is a public link to your CodeCaptain, so you can reach it from your phone or another network. Use `codecaptain tunnel` to create one for a running instance.

## Prerequisites

CodeCaptain starts the tunnel provider CLI on your machine. Install the provider you want to use first:

```bash
brew install cloudflared
brew install ngrok
```

Cloudflare quick tunnels can run with `cloudflared`. Ngrok requires an ngrok account and an authtoken from the ngrok dashboard:

```bash
ngrok config add-authtoken <your-ngrok-token>
```

## Quick start

1. Start CodeCaptain:

```bash
codecaptain
```

If you skip this step, `codecaptain tunnel start` can auto-start a CLI server. When auto-starting, you can pass server options such as `--port`, `--host`, `--lan`, `--ui-password`, and `--api-only`.

2. Start a Cloudflare tunnel:

```bash
codecaptain tunnel start --provider cloudflare --mode quick
```

Or start an Ngrok tunnel:

```bash
codecaptain tunnel start --provider ngrok --mode quick
```

3. Check status:

```bash
codecaptain tunnel status
```

When the tunnel is up, `status` shows a public URL. Open it, or scan the QR code, to reach CodeCaptain from anywhere.

By default, CodeCaptain prints a QR code in interactive TTY sessions. Use `--qr` to force QR output, or `--no-qr` to disable it.

## Providers

- `cloudflare`: quick, managed remote, and managed local modes
- `ngrok`: quick mode

## Managed modes

### Managed remote

Use a token + hostname managed by Cloudflare:

```bash
codecaptain tunnel start --provider cloudflare --mode managed-remote --token-file ~/.secrets/cf-token --hostname app.example.com
```

### Managed local

Use a local `cloudflared` config:

```bash
codecaptain tunnel start --provider cloudflare --mode managed-local --config ~/.cloudflared/config.yml
```

## Profiles (managed-remote)

Save a reusable profile:

```bash
codecaptain tunnel profile add --provider cloudflare --mode managed-remote --name prod-main --hostname app.example.com --token-file ~/.secrets/cf-token
```

Start using the saved profile:

```bash
codecaptain tunnel start --profile prod-main
```

## Useful commands

```bash
codecaptain tunnel providers
codecaptain tunnel ready --provider cloudflare
codecaptain tunnel ready --provider ngrok
codecaptain tunnel doctor --provider cloudflare
codecaptain tunnel doctor --provider ngrok
codecaptain tunnel stop --port 3000
```

## Behavior notes

- one active tunnel per CodeCaptain instance (port)
- starting a new mode/provider on same instance replaces previous tunnel
- generating a new connect link revokes previous unused one
- tunnel auto-start preserves server flags like `--ui-password` and `--api-only` in the instance settings used by restart/update flows
