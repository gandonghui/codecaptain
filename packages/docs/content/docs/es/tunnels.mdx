# Túneles

Un túnel es un enlace público a tu CodeCaptain, para que puedas acceder a él desde tu teléfono u otra red. Usa `codecaptain tunnel` para crear uno para una instancia en marcha.

## Requisitos previos

CodeCaptain inicia el CLI del proveedor de túnel en tu máquina. Instala primero el proveedor que quieras usar:

```bash
brew install cloudflared
brew install ngrok
```

Los túneles rápidos de Cloudflare pueden funcionar con `cloudflared`. Ngrok requiere una cuenta de ngrok y un authtoken del panel de ngrok:

```bash
ngrok config add-authtoken <your-ngrok-token>
```

## Inicio rápido

1. Inicia CodeCaptain:

```bash
codecaptain
```

Si omites este paso, `codecaptain tunnel start` puede iniciar automáticamente un servidor CLI. Al hacerlo, puedes pasar opciones del servidor como `--port`, `--host`, `--lan`, `--ui-password` y `--api-only`.

2. Inicia un túnel de Cloudflare:

```bash
codecaptain tunnel start --provider cloudflare --mode quick
```

O inicia un túnel de Ngrok:

```bash
codecaptain tunnel start --provider ngrok --mode quick
```

3. Comprueba el estado:

```bash
codecaptain tunnel status
```

Cuando el túnel está activo, `status` muestra una URL pública. Ábrela, o escanea el código QR, para acceder a CodeCaptain desde cualquier lugar.

Por defecto, CodeCaptain imprime un código QR en sesiones TTY interactivas. Usa `--qr` para forzar la salida del QR, o `--no-qr` para desactivarla.

## Proveedores

- `cloudflare`: modos quick, managed remote y managed local
- `ngrok`: modo quick

## Modos gestionados

### Gestionado remoto

Usa un token + nombre de host gestionados por Cloudflare:

```bash
codecaptain tunnel start --provider cloudflare --mode managed-remote --token-file ~/.secrets/cf-token --hostname app.example.com
```

### Gestionado local

Usa una configuración local de `cloudflared`:

```bash
codecaptain tunnel start --provider cloudflare --mode managed-local --config ~/.cloudflared/config.yml
```

## Perfiles (managed-remote)

Guarda un perfil reutilizable:

```bash
codecaptain tunnel profile add --provider cloudflare --mode managed-remote --name prod-main --hostname app.example.com --token-file ~/.secrets/cf-token
```

Inicia usando el perfil guardado:

```bash
codecaptain tunnel start --profile prod-main
```

## Comandos útiles

```bash
codecaptain tunnel providers
codecaptain tunnel ready --provider cloudflare
codecaptain tunnel ready --provider ngrok
codecaptain tunnel doctor --provider cloudflare
codecaptain tunnel doctor --provider ngrok
codecaptain tunnel stop --port 3000
```

## Notas de comportamiento

- un único túnel activo por instancia de CodeCaptain (puerto)
- iniciar un nuevo modo/proveedor en la misma instancia reemplaza el túnel anterior
- generar un nuevo enlace de conexión revoca el anterior sin usar
- el autoarranque del túnel conserva flags del servidor como `--ui-password` y `--api-only` en los ajustes de instancia usados por reinicios y actualizaciones
