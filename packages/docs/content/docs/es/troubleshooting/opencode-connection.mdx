# Conexión de CodeCaptain-core

Si CodeCaptain carga pero nunca pasa de "CodeCaptain-core is restarting", o el chat no responde, el servidor con el que se comunica no es accesible. Repasa estos pasos.

## Se queda atascado en "CodeCaptain-core is restarting"

- dale un momento justo después de iniciar: este estado es normal mientras el servidor arranca
- comprueba si el servidor está vivo con `codecaptain status`
- reinícialo con `codecaptain restart`
- ve los detalles de arranque con `codecaptain logs`

## Te conectas a tu propio servidor

Si configuraste CodeCaptain para usar un servidor existente, revisa de nuevo la configuración de [CodeCaptain-core Server](/es/opencode-server/):

- `OPENCODE_HOST` debe incluir el puerto, sin ruta; por ejemplo `http://localhost:4096`
- define `OPENCODE_SKIP_START=true` para que CodeCaptain no inicie también el suyo
- si la dirección es inválida, CodeCaptain la ignora e inicia su propio servidor en su lugar; busca una advertencia `[config]` en los registros

## Sigue fallando

- confirma que Node.js sea la versión `20` o más reciente
- reinstala la última CLI
- abre `http://localhost:3000` directamente antes de probar cualquier túnel o enlace remoto
