# 환경 변수

CodeCaptain는 시작할 때 이 변수들을 읽습니다. 시작 서비스는 `codecaptain startup enable` 실행 시 기본적으로 현재 환경의 snapshot을 저장하므로, 서비스에 적용할 변수를 바꾼 뒤에는 다시 실행하세요.

## CodeCaptain 서버

### `CODECAPTAIN_HOST`

CodeCaptain 웹 서버가 바인딩할 주소입니다. 다른 컴퓨터에서 접속해야 하면 `0.0.0.0`을 사용하세요.

### `CODECAPTAIN_UI_PASSWORD`

브라우저 UI 비밀번호입니다. localhost 밖으로 바인딩하거나 터널, 리버스 프록시를 사용할 때 설정하세요.

### `CODECAPTAIN_API_ONLY`

`true` 또는 `1`이면 CodeCaptain를 headless 모드로 시작합니다. 데스크톱과 모바일 클라이언트용 API route는 계속 사용할 수 있지만 브라우저 UI는 제공하지 않습니다.

### `CODECAPTAIN_DATA_DIR`

CodeCaptain 데이터 디렉터리를 바꿉니다. 기본값은 `~/.config/codecaptain`입니다.

### `CODECAPTAIN_COMPRESS_API`

API 응답 압축을 제어합니다. `true` 또는 `1`은 켜고, `false` 또는 `0`은 끕니다.

### `CODECAPTAIN_SKIP_API_COMPRESSION`

`true` 또는 `1`이면 API 응답 압축을 끕니다. `CODECAPTAIN_COMPRESS_API`보다 우선합니다.

### `CODECAPTAIN_VERBOSE_REQUEST_LOGS`

`true` 또는 `1`이면 자세한 HTTP request logs를 켭니다.

### `CODECAPTAIN_UPDATE_API_URL`

업데이트 확인 API endpoint를 바꿉니다. 대부분의 사용자는 설정하지 않아도 됩니다.

### `CODECAPTAIN_PACKAGE_MANAGER`

자동 감지가 틀렸을 때 업데이트에 사용할 package manager를 강제로 지정합니다.

## CodeCaptain-core 서버

### `OPENCODE_HOST`

CodeCaptain를 이미 실행 중인 CodeCaptain-core server에 연결합니다. 값은 명시적 포트가 있고 path, query, hash가 없는 `http` 또는 `https` origin이어야 합니다. `OPENCODE_HOST`는 `OPENCODE_PORT`보다 우선합니다.

### `OPENCODE_PORT`

CodeCaptain-core server 포트를 설정합니다. managed CodeCaptain-core에서는 이 포트를 요청하고, `OPENCODE_SKIP_START=true`와 함께 쓰면 해당 포트의 외부 서버에 연결합니다.

### `OPENCODE_SKIP_START`

`true`이면 CodeCaptain가 자체 CodeCaptain-core server를 시작하지 않습니다.

### `CODECAPTAIN_OPENCODE_HOSTNAME`

CodeCaptain가 관리하는 CodeCaptain-core server의 bind hostname입니다. 기본값은 `127.0.0.1`입니다.

### `OPENCODE_BINARY`

CodeCaptain가 실행할 `opencode` executable 경로입니다.

### `OPENCODE_CONFIG`

특정 CodeCaptain-core config 파일 경로입니다.

### `OPENCODE_CONFIG_DIR`

agents, skills, snippets, config discovery에 사용할 CodeCaptain-core config 디렉터리입니다.

### `OPENCODE_DATA_DIR`

managed CodeCaptain-core server의 사용자 지정 데이터 디렉터리입니다.

### `OPENCODE_WSL_DISTRO`

Windows에서 CodeCaptain-core 연동에 사용할 WSL distro를 선택합니다.

### `CODECAPTAIN_OPENCODE_WSL_DISTRO`

WSL distro를 선택하는 CodeCaptain 전용 alias입니다. 둘 다 설정하면 `OPENCODE_WSL_DISTRO`가 우선합니다.

### `OPENCODE_JWT_SECRET`

UI auth tokens 서명에 사용하는 secret입니다. 지속 실행 서비스에서는 긴 무작위 값을 사용하세요.

## Terminal 및 Git

### `CODECAPTAIN_TERMINAL_SHELL`

CodeCaptain terminal sessions에서 사용할 shell executable입니다.

### `CODECAPTAIN_GIT_BINARY`

CodeCaptain Git 기능에서 사용할 Git executable입니다.

### `GIT_BINARY`

Git executable의 대체 override입니다. CodeCaptain 전용 설정에는 `CODECAPTAIN_GIT_BINARY`를 권장합니다.

### `CODECAPTAIN_GIT_READ_CACHE_TTL_MS`

Git-backed file reads cache의 TTL이며 단위는 밀리초입니다. 디버깅 중 cache를 끄려면 `0`으로 설정하세요.

## 음성 및 터널

### `OPENAI_API_KEY`

OpenAI-compatible services를 호출하는 CodeCaptain 음성 기능에 사용하는 API key입니다.

### `CODECAPTAIN_ALLOW_REMOTE_OPENAI_COMPAT_URLS`

`true` 또는 `1`이면 음성 기능에서 원격 OpenAI-compatible base URLs를 허용합니다.

### `NGROK_AUTHTOKEN`

CodeCaptain tunnel commands에서 사용하는 ngrok auth token입니다. `ngrok config add-authtoken <token>`으로도 설정할 수 있습니다.

## Runtime helpers

### `BUN_BINARY`

CodeCaptain가 daemon processes를 시작할 때 사용할 Bun executable입니다.

### `BUN_INSTALL`

Bun 설치 루트입니다. CodeCaptain는 daemon 시작과 업데이트를 위해 여기서 `bin/bun`을 찾습니다.

### `VITE_OPENCODE_URL`

Vite-built web app의 build-time API base URL입니다. 일반 CLI 또는 desktop 사용에서는 대부분 설정할 필요가 없습니다.
